# Proceso de validación usuario plataforma

Todos los procesos tienen un evento  

El evento es el que gatilla o el que empieza a hacer la operación de inicio del proceso.  

Todo proceso tiene una entrada, operaciones o tareas y tiene una salida.  


### Entradas
- Login y Password
- Rol (Estudiante, docente, etc)

### Operaciones
- Mostrar la interfaz que me permite ingresar los datos
- Recordar inicio, recuperar contraseña. (En un caso de uso, recuperar contraseña es un extend)

### Salidas
- Un mensaje sobre la validación de los campos.
- Un mensaje sobre que tienen formato incorrecto.
- Redireccionar a mi perfil

# Elementos
- Evento
- Tareas
- Compuertas
- Conectores (Secuencia, mensaje ,asociación)
- Pool =>
- Lane

> [!IMPORTANT]
> Al tener un pool con lanes (subdividido) es porque el proceso lo controlo yo. En el caso de usuario => biblioteca, tiene que ser en pools separadas porque no controlo al usuario

> [!IMPORTANT]
> En un mismo pool distinto lane, las unicas lineas son secuencia y asociación.

> [!IMPORTANT]
> La linea de mensaje solo se utiliza en coreografía cuando son pools diferentes.

> [!IMPORTANT]
> En orquestación todas las tareas estan relacionadas.
